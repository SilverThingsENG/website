What is the Hypertext Transfer Protocol (HTTP)

An application-layer protocol for transmitting hypermedia documents, such as HTML.
Designed for communication between web browsers and web servers.

Why is HTTP called a stateless protocol?

There is no link between two requests being successively carried out on the same connection.

Describe the three phases of establishing a session using client-server protocols such as HTTP

Phase 1. The client establishes a TCP connection.
Phase 2. The client sends its request e.g. Get, Fetch, Post, etc
Phase 3. The server processed the request, sending back its answer and providing a status code and the data required.

Describe the parameters on a get and post request.

Get Request: Requests using 'Get' should only retrieve data.
Post Request: Post would only send data to a server, generally in the form of an HTML form.

What are the HTTP response status code and What do the HTTP status codes, 200, and 404 indicate?

Status codes indicated if a request was successful or not and why.
200 means that a request was successful.
400 means that the pages that was requested by the client wasn't found.

What are HTTP cookies? Give an example.

HTTP cookies are small pieces of data that a server sends to the user's browser. Typically used to see if two requests came from the same browser.
Examples:
Session Management - Keeping a user logged in, remembering shopping carts etc.
Personalisation: User preferences, themes, and other user defined settings.
Tracking: Recording and analysing a user's behaviour.

Why are cookies a security issue?

They contain personal information that can be easily hijacked and the information that is stored within can be taken and used.
This will allow the interceptor to take on a session of a user that is logged into a specific site and pretend to be the victim.

Describe session hijacking and XSS

Cookies are used to identify a user and their session that they may be logged into a website.
Stealing a cookie will allow the attacker to hijack the user's authenticated session.
Ways to retrieve a cookie is through 'Social Engineering', exploiting a person's vulnerability and lack of knowledge, or exploiting a XSS vulnerability, allows an attacker to inject malicious client-side code to retrieve a cookie.

Describe Cross-site request forgery(CSRF)

An attack the impersonates a trusted user and sends a website unwanted commands. THis can be done by including malicious parameters in a URL behind a link that purports to go somewhere else.

What are Cross-site HTTP requests (CORS) and what requests use CORS?

A mechanism that uses additional HTTP headers to tell browsers to give a web application running at one origin, access to selected resources at a different origin.
Requests that can use CORS: Web Fonts, Images/video frames drawn from canvas, CSS Shapes from Images.
